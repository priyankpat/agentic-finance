💡 PRO TIPS
════════════════════════════════════════════════════════════════════════════

1. Start Small
   Don't implement everything at once. Week 1 = data + categorization.

2. Use Haiku First
   60% of tasks use Haiku ($0.0008). Escalate only if needed.

3. Batch Aggressively
   Send 10-20 items per call = 90% cost savings vs individual calls.

4. Cache Everything
   5-min TTL on prices = 80% lookup savings.

5. Monitor Costs Daily
   Know exactly where money goes. Weekly review mandatory.

6. Test Prompts
   Better prompt = better results at same cost.

7. Use Rules First
   70% of transactions match simple rules (free!).

8. Measure Everything
   Track costs, accuracy, time saved, ROI.

⚠️ AVOID THESE MISTAKES
════════════════════════════════════════════════════════════════════════════

❌ Using Opus for everything (37.5× more expensive than Haiku)
❌ Not batching requests (10× more expensive than batched)
❌ Verbose prompts (25× more expensive than concise)
❌ High max_tokens unnecessarily (75% wasted output tokens)
❌ No caching (80% wasted lookups)
❌ Not monitoring costs (costs creep up silently)
❌ Perfectionism (94% is good enough, ship it)
