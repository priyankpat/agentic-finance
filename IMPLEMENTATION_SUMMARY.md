## 💡 Pro Tips

### 1. Start Small
Don't try to implement everything at once. Start with Week 1 (data + categorization).

### 2. Measure Everything
Track costs from day 1. Review them weekly. This is non-negotiable.

### 3. Use Rules First
70% of transactions can be categorized with simple rules (free!).

### 4. Batch Aggressively
Send 10-20 transactions per API call. This is huge for cost efficiency.

### 5. Cache Everything
Prices change every 5 minutes. Cache for 5 minutes. Saves 80% on lookups.

### 6. Test Your Prompts
Spend time on prompts. A better prompt = better results at same cost.

### 7. Monitor Costs Daily
Use the cost tracking code. Know exactly where money is going.

### 8. Start with Haiku
Use cheapest model first. Only escalate to Sonnet/Opus if needed.

## 🚨 Common Pitfalls to Avoid

❌ **Pitfall 1**: Using Opus for everything
- Problem: 37.5× more expensive than Haiku
- Solution: Use decision tree, start with Haiku

❌ **Pitfall 2**: Not batching
- Problem: 10 calls cost 10× more than 1 batched call
- Solution: Implement batching from day 1

❌ **Pitfall 3**: Verbose prompts
- Problem: 500 tokens = 25× cost of 20 tokens
- Solution: Be concise, use templates

❌ **Pitfall 4**: No caching
- Problem: Fetch same data repeatedly, waste money
- Solution: Cache with 5-min TTL minimum

❌ **Pitfall 5**: Not monitoring costs
- Problem: Costs creep up, you don't notice
- Solution: Weekly cost review in calendar

❌ **Pitfall 6**: Perfectionism
- Problem: Waiting for perfect accuracy (98.5% vs 94%)
- Solution: 94% is good enough, ship it, iterate

❌ **Pitfall 7**: Over-engineering
- Problem: Building features nobody needs
- Solution: MVP first, features after
